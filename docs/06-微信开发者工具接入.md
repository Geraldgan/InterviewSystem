# 微信开发者工具接入

这份文档专门解决“把当前 uni-app 小程序项目接进微信开发者工具”。

## 1. 先确认后端是通的

当前建议使用的后端地址是：

`http://10.0.20.45:8081`

这是你当前这台 Mac 在同一个 Wi‑Fi 下的局域网地址。

先启动后端：

```bash
cd /Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/backend
SERVER_PORT=8081 JAVA_TOOL_OPTIONS='-Djavax.net.ssl.trustStoreType=KeychainStore -Djavax.net.ssl.trustStore=NONE' ./gradlew bootRun
```

然后在你电脑浏览器访问：

- [http://10.0.20.45:8081/api/health](http://10.0.20.45:8081/api/health)

如果能看到：

```json
{"status":"ok"}
```

说明后端没问题。

## 2. 当前前端接口地址已经配好

文件：

- [frontend/.env.local](/Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/frontend/.env.local)

内容已经是：

```bash
VITE_API_BASE_URL=http://10.0.20.45:8081
```

所以你现在不需要再手改前端请求地址。

## 3. 生成微信小程序构建产物

进入前端目录：

```bash
cd /Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/frontend
```

### 开发模式

```bash
npm run dev:mp-weixin
```

特点：

- 适合你边改代码边看结果
- 终端需要保持运行
- 微信开发者工具能看到最新产物

### 一次性构建

```bash
npm run build:mp-weixin
```

特点：

- 适合先导入看看
- 不需要一直挂着终端

## 4. 小程序构建目录在哪里

通常会在下面两个目录之一：

- 开发模式：`frontend/dist/dev/mp-weixin`
- 构建模式：`frontend/dist/build/mp-weixin`

你导入微信开发者工具时，就选这个目录。

## 5. manifest 里 AppID 现在是什么状态

文件：

- [manifest.json](/Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/frontend/src/manifest.json)

当前：

```json
"mp-weixin": {
  "appid": ""
}
```

这说明现在是“先本地开发联调”的状态。

你有两种选择：

1. 你已经有自己的小程序 AppID  
把它填进 `manifest.json` 的 `mp-weixin.appid`

2. 你还没有正式 AppID  
当前导出产物已经自动生成了 `touristappid`，可以先直接导入开发调试

对应文件：

- [project.config.json](/Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/frontend/dist/build/mp-weixin/project.config.json)

## 6. 微信开发者工具里怎么导入

1. 打开微信开发者工具
2. 选择“导入项目”
3. 项目目录选择：

```text
/Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/frontend/dist/dev/mp-weixin
```

或者：

```text
/Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/frontend/dist/build/mp-weixin
```

4. 如果你有自己的 AppID，就填自己的
5. 如果你只是本地调试，当前导出产物可以直接以测试模式导入
6. 点击导入

## 7. 导入后第一件事要做什么

我已经确认当前导出的 [project.config.json](/Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/frontend/dist/build/mp-weixin/project.config.json) 里：

- `appid` 是 `touristappid`
- `setting.urlCheck` 是 `false`

这代表本地导入时已经比默认状态更适合联调。

不过为了稳妥起见，你仍然可以在开发者工具中再确认一次：

`详情 / 本地设置 / 不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书`

## 8. 页面加载后你应该看到什么

首页应该会展示：

- AI Interview Studio
- 岗位模板卡片
- 岗位名称、难度、题目数量、知识点输入框

如果岗位模板没出来，优先看：

1. 后端是否还在运行
2. `.env.local` 的 IP 是否还是 `10.0.20.45`
3. 你的手机和电脑是否还在同一个 Wi‑Fi
4. 微信开发者工具是否勾选了“不校验合法域名”

## 9. 点“生成题集”后应该发生什么

点击“生成题集”后，前端会调用：

- `POST /api/question-sets/generate`

成功后会自动跳到详情页，看到：

- 题集摘要
- 每道题
- 对应答题思路

## 11. 你现在最推荐的联调顺序

1. 启动 Spring Boot 到 `8081`
2. 在电脑浏览器访问 `http://10.0.20.45:8081/api/health`
3. 在前端执行 `npm run dev:mp-weixin`
4. 打开微信开发者工具导入 `frontend/dist/dev/mp-weixin`
5. 关闭合法域名校验
6. 先看首页岗位是否正常加载
7. 再测试“生成题集”

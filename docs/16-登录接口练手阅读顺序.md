# 登录接口练手阅读顺序

这份文档是给你做“登录接口练手导航”的。

如果你现在想系统练一次：

- 设计接口
- 分层写代码
- 补密码校验
- 测试接口
- 最后自检复盘

那建议你按下面这 4 份文档的顺序来。

## 为什么是这个顺序

这个顺序不是随便排的，而是按“从抽象到落地，再到验收”的顺序来的。

也就是：

```text
先看路线
-> 再搭骨架
-> 再补关键实现和测试
-> 最后做验收
```

这样你不容易出现这些情况：

- 一开始就写细节，结果整体结构没想清楚
- 写完骨架后，不知道密码和 token 该怎么落地
- 代码写完了，却没有系统自检

---

## 你边读边做时建议怎么推进

建议你按下面这个节奏。

### 读第 1 份时

你只做设计，不急着写实现：

- 想清楚接口路径
- 想清楚请求体
- 想清楚响应体
- 想清楚用户表字段
- 想清楚登录主流程

### 读第 2 份时

你开始搭代码骨架：

- 建实体
- 建 Repository
- 建 DTO
- 建 Service
- 建 Controller

### 读第 3 份时

你开始补最关键的实现：

- 密码匹配
- token 生成
- 测试用户准备
- `curl` 联调

### 读第 4 份时

你开始做验收和复盘：

- 成功场景检查
- 失败场景检查
- 安全点检查
- 分层检查

---

## 这套练手读完以后，你应该收获什么

如果你真的按顺序做完这 4 份文档，你应该至少会比现在更稳地掌握这些东西：

- 接口开发不是只写一个 Controller 方法
- 一个登录接口至少要经过 `Entity / Repository / DTO / Service / Controller`
- 密码不能明文存数据库
- 登录逻辑的核心应该写在 Service
- Controller 应该尽量保持轻
- 第一版接口写完后必须做成功 / 失败场景验证

如果这些你都已经能说清楚，说明这次练手非常值。

## 练完登录接口后推荐你接着练什么

如果你完成了这次登录接口练手，下一步最值得接着练的接口是：

### 1. 注册接口

因为它正好和登录接口配套，能帮你练：

- 参数校验
- 用户唯一性校验
- 密码编码后入库

### 2. 获取当前用户信息接口

因为它能帮你练：

- token 解析
- 当前用户识别
- 简单鉴权思路

### 3. 修改密码接口

因为它能帮你练：

- 旧密码校验
- 新密码编码
- 安全逻辑完整性

也就是说，登录接口练完以后，你最适合继续练“账号体系”的另外两个接口。

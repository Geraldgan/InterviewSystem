# 后端从零搭建

这份文档按“iOS 工程师首次接触 Spring Boot”的视角来写。

## 1. 后端目录怎么理解

`backend/src/main/java/com/interview/backend/` 下最重要的目录：

- `controller/`: 对外暴露接口，类似 iOS 里的网络入口
- `service/`: 业务逻辑层，负责真正“做事”
- `entity/`: 数据库表对应的实体
- `repository/`: 数据访问层，负责查表和存表
- `config/`: 配置项与基础设施
- `support/`: Prompt、生成命令、内部结果对象
- `exception/`: 统一错误处理

## 2. 为什么选 H2

你要求“数据存本地数据库”，而且当前阶段主要是学习与验证闭环，所以我选了 H2 文件数据库：

- 配置简单
- 不需要额外安装 MySQL
- 启动项目后就能直接存数据
- 后续换 MySQL / PostgreSQL 也比较容易

数据库文件默认在：

`backend/data/interview-system.mv.db`

主要由这几个类协作：

- [QuestionSetController.java](/Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/backend/src/main/java/com/interview/backend/controller/QuestionSetController.java)
- [QuestionSetService.java](/Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/backend/src/main/java/com/interview/backend/service/QuestionSetService.java)
- [QuestionGenerationService.java](/Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/backend/src/main/java/com/interview/backend/service/QuestionGenerationService.java)
- [OpenAiQuestionGenerator.java](/Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/backend/src/main/java/com/interview/backend/service/OpenAiQuestionGenerator.java)

调用顺序：

1. Controller 接收 HTTP 请求
2. Service 解析岗位和参数
3. PromptBuilder 生成 Prompt 与 JSON Schema
4. OpenAI 返回结构化题集
5. Service 把题目落库
6. Controller 返回给前端

## 4. OpenAI 配置怎么生效

配置文件在：

- [application.yml](/Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/backend/src/main/resources/application.yml)

环境变量说明：

- `OPENAI_API_KEY`: 真实调用 OpenAI 的 Key
- `OPENAI_MODEL`: 默认是 `gpt-5-mini`
- `OPENAI_REASONING_EFFORT`: 默认是 `low`

如果没有配置 `OPENAI_API_KEY`：

- 系统会进入本地模拟生成器
- 依然能返回结构化题集
- 数据照样会落到数据库里

## 5. 为什么默认用 `gpt-5-mini`

当前项目生成的是“结构化面试题 + 答题思路”，它属于质量要稳定、但也要控制成本的场景，所以我默认选了更轻量的模型。

如果你后面更重视质量，可以把：

```bash
OPENAI_MODEL=gpt-5.2
```

或者替换成你确认后的其他新模型。

## 6. 启动命令

```bash
cd /Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/backend
export OPENAI_API_KEY="你的 OpenAI Key"
JAVA_TOOL_OPTIONS='-Djavax.net.ssl.trustStoreType=KeychainStore -Djavax.net.ssl.trustStore=NONE' ./gradlew bootRun
```

之所以带上 `JAVA_TOOL_OPTIONS`，是因为当前这台 macOS 上 Gradle 访问外部依赖时需要显式使用系统证书库。

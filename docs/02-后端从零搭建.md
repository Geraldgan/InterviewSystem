# 后端从零搭建

这份文档按“iOS 工程师首次接触 Spring Boot”的视角来写。

## 1. 后端目录怎么理解

`backend/src/main/java/com/interview/backend/` 下最重要的目录：

- `controller/`: 对外暴露接口，类似 iOS 里的网络入口
- `service/`: 业务逻辑层，负责真正“做事”
- `entity/`: 数据库表对应的实体
- `repository/`: 数据访问层，负责查表和存表
- `config/`: 配置项与基础设施
- `support/`: Prompt、生成命令、内部结果对象
- `exception/`: 统一错误处理

## 2. 为什么默认改成 MySQL

当前项目已经把默认持久化数据库切到 MySQL，主要是为了让本地开发环境更接近真实部署：

- 多端联调时和线上数据库行为更一致
- 后续迁移服务器、做备份、加索引会更自然
- Spring Data JPA 在 MySQL 上的行为更贴近实际生产场景

建议先准备一个本地数据库：

```bash
mysql -uroot -p -e "CREATE DATABASE IF NOT EXISTS interview_system DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
```

后端默认读取下面几个环境变量：

- `SPRING_DATASOURCE_URL`
- `SPRING_DATASOURCE_USERNAME`
- `SPRING_DATASOURCE_PASSWORD`

测试环境仍然保留 H2 内存库，并开启 MySQL 兼容模式，这样本地跑测试时不依赖 MySQL 服务。

调用顺序：

1. Controller 接收 HTTP 请求
2. Service 解析岗位和参数
3. PromptBuilder 生成 Prompt 与 JSON Schema
4. OpenAI 返回结构化题集
5. Service 把题目落库
6. Controller 返回给前端

## 4. OpenAI 配置怎么生效

配置文件在：

- [application.yml](/Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/backend/src/main/resources/application.yml)

环境变量说明：

- `SPRING_DATASOURCE_URL`: MySQL 连接串
- `SPRING_DATASOURCE_USERNAME`: MySQL 用户名
- `SPRING_DATASOURCE_PASSWORD`: MySQL 密码
- `OPENAI_API_KEY`: 真实调用 OpenAI 的 Key
- `OPENAI_MODEL`: 默认是 `gpt-5-mini`
- `OPENAI_REASONING_EFFORT`: 默认是 `low`

如果没有配置 `OPENAI_API_KEY`：

- 系统会进入本地模拟生成器
- 依然能返回结构化题集
- 数据照样会落到 MySQL 数据库里

## 5. 为什么默认用 `gpt-5-mini`

当前项目生成的是“结构化面试题 + 答题思路”，它属于质量要稳定、但也要控制成本的场景，所以我默认选了更轻量的模型。

如果你后面更重视质量，可以把：

```bash
OPENAI_MODEL=gpt-5.2
```

或者替换成你确认后的其他新模型。

## 6. 启动命令

```bash
cd /Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/backend
export SPRING_DATASOURCE_URL="jdbc:mysql://127.0.0.1:3306/interview_system?createDatabaseIfNotExist=true&useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=Asia/Shanghai&characterEncoding=UTF-8"
export SPRING_DATASOURCE_USERNAME="root"
export SPRING_DATASOURCE_PASSWORD="你的 MySQL 密码"
export OPENAI_API_KEY="你的 OpenAI Key"
JAVA_TOOL_OPTIONS='-Djavax.net.ssl.trustStoreType=KeychainStore -Djavax.net.ssl.trustStore=NONE' ./gradlew bootRun
```

之所以带上 `JAVA_TOOL_OPTIONS`，是因为当前这台 macOS 上 Gradle 访问外部依赖时需要显式使用系统证书库。

# 网站开发与运行教程

这份教程专门讲 `H5 网站` 这一端怎么开发、怎么跑、怎么调试。

如果你是从 `iOS + SwiftUI` 视角切过来，可以把它理解成：

- `frontend/` 就像网站客户端工程
- `backend/` 就像统一的数据服务层
- 浏览器相当于 iPhone 模拟器，只不过运行的是 `uni-app` 打出来的 `H5`

## 1. 先理解网站这一端在整个项目里的位置

当前项目是一个 `API-first` 架构：

- `backend/`：`Spring Boot` 后端，负责岗位模板、AI 出题、题集落库
- `frontend/`：`uni-app + Vue 3` 前端，负责 `Web / 微信小程序 / Android`
- `ios-swiftui/`：原生 `SwiftUI` iOS 客户端

网站端的请求链路很简单：

1. 浏览器打开 `frontend/` 跑起来的 H5 页面
2. 页面里的表单调用前端 API
3. 前端 API 请求 `Spring Boot` 后端
4. 后端返回岗位列表、题集详情、AI 生成结果

## 3. 第一次跑起来，按这个顺序做

### 第 1 步：启动后端

先在一个终端里启动 `Spring Boot`：

```bash
cd /Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/backend
export SPRING_DATASOURCE_URL="jdbc:mysql://127.0.0.1:3306/interview_system?createDatabaseIfNotExist=true&useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=Asia/Shanghai&characterEncoding=UTF-8"
export SPRING_DATASOURCE_USERNAME="root"
export SPRING_DATASOURCE_PASSWORD="你的 MySQL 密码"
export OPENAI_API_KEY="你的 OpenAI Key"
SERVER_PORT=8081 JAVA_TOOL_OPTIONS='-Djavax.net.ssl.trustStoreType=KeychainStore -Djavax.net.ssl.trustStore=NONE' ./gradlew bootRun
```

说明：

- `SERVER_PORT=8081` 是我们当前本地联调用的端口
- 启动前要先保证本地 MySQL 已经启动，并且 `interview_system` 数据库可访问
- 如果你暂时没配 `OPENAI_API_KEY`，系统也能运行，会自动回退到本地模拟题生成器

启动后可以在另一个终端检查健康接口：

```bash
curl -s http://127.0.0.1:8081/api/health
```

如果返回：

```json
{"status":"ok"}
```

说明后端已经通了。

### 第 2 步：确认前端环境变量

前端接口地址通过这个文件控制：

- [frontend/.env.local](/Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/frontend/.env.local)

当前文件已经帮你配好了：

```bash
VITE_API_BASE_URL=http://10.0.20.45:8081
```

这套配置更适合：

- 微信小程序真机调试
- 手机和电脑在同一 Wi-Fi 下联调

如果你现在只是想在 `Mac 浏览器` 里调试网站，建议临时改成：

```bash
VITE_API_BASE_URL=http://127.0.0.1:8081
```

原因是：

- `127.0.0.1` 对你自己的电脑浏览器最直接
- 局域网 IP 更适合给手机、真机、小程序访问

### 第 3 步：安装前端依赖

```bash
cd /Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/frontend
npm install
```

### 第 4 步：启动网站开发服务

```bash
npm run dev:h5
```

正常情况下，终端会输出本地调试地址。你直接用浏览器打开它即可。

如果你想显式指定地址和端口，可以用：


这个命令的意思是：

- `--port 5173`：固定开发端口

### 第 5 步：浏览器里验证一遍核心链路

进入首页后，按这个顺序验证：

1. 页面能否正常打开
2. 岗位模板是否能加载出来
3. 输入补充要求后，点击“生成题集”
4. 能否跳转到详情页

如果这 4 步都通了，说明网站端联调已经成功。

## 4. 现在这个网站的开发流程是什么

1. 改页面结构或输入项
2. 改前端请求参数
3. 改后端 DTO / Service / Controller
4. 本地联调
5. 构建产物

你可以把它拆成下面这套固定动作。

### 场景 1：只改页面文案、布局、样式

改完后刷新浏览器，看页面变化即可。

### 场景 2：新增一个“出题参数”

比如你想新增一个“是否偏重项目实战”的字段，建议按这个顺序做：

1. 在 [home/index.vue](/Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/frontend/src/pages/home/index.vue) 增加表单字段
2. 在 `handleGenerate()` 里把参数带进请求体
3. 如果需要复用接口方法，就同步看 [interview.js](/Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/frontend/src/api/interview.js)
4. 到后端修改生成请求 DTO 和业务逻辑
5. 重启后端或等待热更新
6. 回到浏览器重新测试生成题集

### 场景 3：新增一个页面

比如你想做“收藏题目”页面，步骤通常是：

1. 新建一个页面目录，例如 `frontend/src/pages/favorite/index.vue`
2. 在 [pages.json](/Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/frontend/src/pages.json) 注册页面
3. 如果要进 TabBar，再同步修改 `tabBar`
4. 如果页面需要后端数据，再补充 `api/` 请求封装

## 5. 网站这一端的代码是怎么串起来的

下面这条链路非常重要，你把它看顺了，后面改功能会轻松很多。

### 首页加载岗位模板

入口文件：

- [home/index.vue](/Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/frontend/src/pages/home/index.vue)

关键方法：

- `onLoad()`：页面加载时触发
- `loadPositions()`：请求岗位模板列表
- `applyProfile()`：把岗位模板回填到表单

实际调用关系：

1. `onLoad()` 调用 `loadPositions()`
2. `loadPositions()` 调用 `fetchPositions()`
3. `fetchPositions()` 在 [interview.js](/Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/frontend/src/api/interview.js) 里继续调用 `request('/api/positions')`
4. `request()` 在 [http.js](/Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/frontend/src/api/http.js) 里真正发出网络请求

### 点击“生成题集”

首页里的核心方法是：

- `handleGenerate()`

它会做这些事：

1. 从表单收集参数
2. 把关注知识点切成数组
3. 调用 `generateQuestionSet(payload)`
4. 请求后端 `POST /api/question-sets/generate`
5. 成功后跳转到详情页

这部分是你最值得先读懂的代码。

## 6. 日常开发时建议怎么调试

### 浏览器里先看 Network

当你遇到“页面能开，但是数据不对”时，第一步先开浏览器开发者工具：

1. 打开网站页面
2. 打开 `Network`
3. 刷新页面
4. 观察这几个接口

- `/api/health`
- `/api/positions`
- `/api/question-sets`
- `/api/question-sets/generate`

你重点看三件事：

- 请求 URL 对不对
- 返回状态码是不是 `200`
- 返回 JSON 结构是不是你预期的

### 再看前端代码入口

如果接口返回对了，但页面没显示，就按这个顺序查：

1. 页面有没有发请求
2. 请求结果有没有写进 `ref` / `reactive`
3. 模板里有没有正确渲染字段

### 最后看后端日志

如果前端请求发出去了，但接口报错，就去看后端运行终端。

常见现象：

- 参数校验失败
- 请求字段名没对上
- OpenAI 没配 Key
- 数据库写入报错

## 7. 常见报错怎么排

### 1. 浏览器页面能开，但岗位列表加载失败

优先检查：

1. 后端是不是还在运行
2. [frontend/.env.local](/Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/frontend/.env.local) 里的端口是不是 `8081`
3. 浏览器请求的地址是不是你预期的地址

最直接的检查方式：

```bash
curl -s http://127.0.0.1:8081/api/positions
```

如果这里都拿不到数据，先不要看前端，先把后端打通。

### 2. 浏览器报“Could not connect to the server”或 `ERR_CONNECTION_REFUSED`

通常是：

- 后端没启动
- 后端端口不是你前端配置的端口
- 你把 `127.0.0.1` 和局域网 IP 混用了

记忆方式很简单：

- 电脑浏览器访问电脑服务：优先用 `127.0.0.1`
- 手机、小程序访问电脑服务：用电脑局域网 IP

### 3. `npm run dev:h5` 报端口监听失败，比如 `listen EPERM`

这通常不是项目代码错了，而是当前终端环境不允许绑定默认端口或默认地址。

你可以改成显式指定：

```bash
npm run dev:h5 -- --host 127.0.0.1 --port 5173
```

如果 `5173` 也被占用，可以换一个端口，例如：

```bash
npm run dev:h5 -- --host 127.0.0.1 --port 5174
```

### 4. 网站能跑，微信小程序却请求失败

大多数情况是因为你还在用：

```bash
VITE_API_BASE_URL=http://127.0.0.1:8081
```

这对浏览器可以，但对手机和小程序不行。  
这时要改成电脑当前 Wi-Fi 下的局域网 IP。

## 8. 发布或交付时怎么构建网站

开发阶段你主要用：

```bash
npm run dev:h5
```

构建发布产物时用：

```bash
npm run build:h5
```

当前构建输出目录是：

- [frontend/dist/build/h5](/Users/geraldgan/Documents/GeraldGan/实践/InterviewSystem/frontend/dist/build/h5)

你可以把这个目录理解成“网站静态产物”。

后续如果你要部署到服务器，通常会把这里面的文件交给：

- `Nginx`
- 静态资源托管平台
- 公司内部前端发布系统
